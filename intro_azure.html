Introducción a Microsoft Azure

Azure es la plataforma de nube de Microsoft. Permite ejecutar aplicaciones, almacenar datos, crear redes, automatizar procesos y consumir servicios avanzados desde centros de datos distribuidos globalmente.

Idea Central

  • Azure ofrece recursos de TI como servicios bajo demanda.
  • Se paga por uso y se escala según la necesidad del negocio.
  • La misma plataforma cubre infraestructura, plataforma y software administrado.

¿Qué es Microsoft Azure?

Microsoft Azure es una plataforma de computación en la nube que pone a disposición servicios de tecnología a través de Internet. En lugar de comprar y operar toda la infraestructura localmente, una organización puede usar capacidad de cómputo, almacenamiento, bases de datos, redes, seguridad, analítica e inteligencia artificial dentro del ecosistema Azure.

Azure no es un solo producto. Es un conjunto amplio de servicios conectados entre sí que permite construir, hospedar, desplegar y administrar soluciones tecnológicas de distintos tamaños.

Azure convierte recursos tradicionales de TI en servicios consumibles bajo demanda, con alcance global y operación flexible.
2

¿Qué ofrece Azure?

Azure cubre necesidades básicas y avanzadas. Puede utilizarse tanto para escenarios simples, como hospedar una aplicación web, como para arquitecturas empresariales con integración, seguridad, monitoreo y alta disponibilidad.

Infraestructura

Recursos base

Máquinas virtuales, redes virtuales, balanceadores, discos y almacenamiento para levantar y operar entornos completos.

Plataforma

Servicios administrados

Bases de datos, aplicaciones web, contenedores, integración y automatización con menos carga operativa para el equipo.

Capacidades avanzadas

Datos e inteligencia

Analítica, inteligencia artificial, monitoreo y gobierno para construir soluciones más completas y escalables.

Azure permite combinar estos servicios en una sola arquitectura. Un mismo proyecto puede usar red, almacenamiento, identidades, funciones, bases de datos y observabilidad dentro de la misma nube.

¿Cómo se trabaja con Azure?

Azure puede administrarse desde distintas interfaces según el perfil del usuario y el nivel de automatización requerido.

Portal

Azure Portal

Interfaz web visual para crear recursos, revisar configuraciones, monitorear servicios y explorar el estado general del entorno.

Comandos

CLI y PowerShell

Alternativas orientadas a automatización, scripts y operaciones repetibles para administradores, ingenieros y equipos DevOps.

Código

Infraestructura como código

Permite definir entornos mediante plantillas y archivos declarativos para desplegar de forma consistente y controlada.

Modelo operativo

Pago por consumo

  • Los recursos se crean cuando se necesitan.
  • Se ajustan conforme crece o baja la demanda.
  • El gasto depende del uso real del servicio.
Escala

Cobertura global

  • Los servicios se despliegan en regiones distribuidas.
  • Se puede acercar la carga a los usuarios o cumplir requerimientos de residencia de datos.
  • La redundancia regional mejora continuidad y resiliencia.

¿Cómo opera Azure? Modelo jerárquico

Azure funciona bajo una estructura jerárquica de organización y control. Cada nivel tiene una responsabilidad distinta y juntos permiten administrar acceso, facturación, organización y despliegue.

1. Cuenta

Account

Es el punto de entrada. Representa al usuario o empresa y define quién puede acceder a Azure. Puede tener una o varias suscripciones.

Piensa en esto como: la identidad que paga y administra.

2. Suscripción

Subscription

Es la unidad clave de operación. Define facturación, límites y control de acceso mediante RBAC. Todos los recursos viven dentro de una suscripción.

Ejemplo: separar `Dev`, `QA` y `Prod` en suscripciones distintas.

Piensa en esto como: la cuenta financiera más el control de acceso.

3. Región

Region

Es la ubicación física donde viven los recursos, por ejemplo `East US`, `West Europe` o `Mexico Central`. Impacta latencia, cumplimiento y disponibilidad.

Piensa en esto como: dónde corre físicamente tu solución.

4. Grupo de recursos

Resource Group

Es un contenedor lógico para agrupar recursos relacionados. Facilita gestión conjunta, organización y lifecycle.

Ejemplos: `rg-app-web-prod`, `rg-data-analytics-dev`.

Piensa en esto como: una carpeta lógica de recursos.

5. Recurso

Resource

Es el elemento operativo real: una VM, una base de datos, una Storage Account, una Function App o un Key Vault.

Cada recurso pertenece a un grupo de recursos, está dentro de una suscripción y se despliega en una región.

Piensa en esto como: el servicio que realmente usas.

Idea Clave

Qué gobierna cada nivel

  • La suscripción controla dinero y acceso.
  • El grupo de recursos organiza.
  • La región define dónde corre.
  • El recurso ejecuta.
Ejemplo Práctico

Despliegue de una app

  • Cuenta: tu empresa.
  • Suscripción: `Producción`.
  • Región: `Mexico Central`.
  • Grupo de recursos: `rg-app-prod`.
  • Recursos: App Service, base de datos y Storage.
Azure organiza sus servicios en una jerarquía donde la suscripción gobierna, los grupos organizan y los recursos ejecutan dentro de una región específica.

Nubes soberanas o nacionales

Azure también tiene entornos aislados para requisitos especiales de soberanía, cumplimiento y residencia de datos. En la documentación oficial se conocen como sovereign clouds o national clouds.

Azure Government (EE. UU.): US Gov Virginia, US Gov Arizona, US Gov Texas, US DoD Central, US DoD East, US Sec East, US Sec West, US Sec West Central.

Idea clave: estas regiones no son simplemente otra ubicación dentro de la nube global; son entornos separados o marcados para requisitos de soberanía, gobierno, defensa, cumplimiento o residencia nacional de datos.

Nota: la documentación de Microsoft Entra identifica hoy como national clouds a Azure Government y Microsoft Azure operated by 21Vianet, y además indica que Azure Germany fue cerrado el 29 de octubre de 2021.

Valor de Azure para las organizaciones

Azure aporta velocidad, elasticidad y estandarización. En vez de esperar la compra e instalación de infraestructura, los equipos pueden aprovisionar servicios rápidamente y concentrarse en entregar valor funcional.

También facilita escenarios híbridos y de modernización gradual, algo especialmente relevante en empresas que no pueden mover todo a la nube de una sola vez.

Agilidad

Tiempo de salida

Menos tiempo en preparación de infraestructura y más tiempo en construir soluciones.

Elasticidad

Escala bajo demanda

Los recursos pueden crecer o reducirse según la carga sin rediseñar toda la plataforma.

Continuidad

Operación resiliente

La arquitectura global y la variedad de servicios favorecen disponibilidad, respaldo y recuperación.

Entender Azure no significa memorizar todos sus servicios, sino comprender que es una plataforma cloud completa para construir, operar y evolucionar soluciones tecnológicas.
