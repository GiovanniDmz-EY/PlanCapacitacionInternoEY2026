Glosario de Términos Azure

Referencia rápida de conceptos frecuentes en Azure. La idea es contar con definiciones simples pero correctas para ubicar términos de gobierno, identidad, red, datos y operación dentro de una arquitectura.

1. Gobierno y organización

CLI

Command-Line Interface. Interfaz por línea de comandos para administrar recursos o ejecutar operaciones sin usar el portal gráfico.

API

Application Programming Interface. Interfaz para que sistemas o aplicaciones se comuniquen entre sí mediante llamadas controladas.

API Gateway

Puerta de entrada para APIs. Recibe solicitudes de clientes y las reenvía al backend aplicando reglas como autenticación, cuotas, transformaciones, cache o monitoreo.

Azure API Management

Servicio PaaS para publicar, proteger, versionar, transformar y monitorear APIs. Se usa para dar gobierno y control centralizado a backends como Functions, App Service, AKS o sistemas externos.

AKS

Azure Kubernetes Service. Servicio administrado de Azure para desplegar y operar clústeres de Kubernetes.

IaaS

Infrastructure as a Service. Modelo donde el proveedor entrega infraestructura base y el usuario administra sistema operativo, middleware y aplicación.

PaaS

Platform as a Service. Modelo donde el proveedor administra la plataforma y el usuario se enfoca principalmente en la aplicación y su configuración.

SaaS

Software as a Service. Modelo donde el software completo es entregado como servicio administrado al usuario final.

SKU

Combinación comercial y técnica que define capacidad, tamaño, límites, nivel de servicio y precio de un recurso.

Azure Resource Manager (ARM)

Capa de administración y despliegue de recursos en Azure. Organiza recursos, dependencias, permisos y despliegues mediante un modelo unificado.

Azure CLI

Herramienta por línea de comandos multiplataforma para administrar recursos Azure desde scripts o automatizaciones.

Azure PowerShell

Conjunto de módulos PowerShell para administrar Azure mediante comandos y scripts orientados al ecosistema PowerShell.

Suscripción

Unidad administrativa y de facturación en Azure. Los recursos se crean dentro de una suscripción y los costos, límites y permisos suelen organizarse a este nivel.

Grupo de recursos

Contenedor lógico donde se agrupan recursos relacionados. Facilita administración, despliegue, permisos y eliminación conjunta de componentes de una solución.

Recurso

Cualquier servicio o componente creado en Azure, por ejemplo una VM, una base de datos, una Storage Account o una VNet.

Etiqueta (Tag)

Par clave-valor que se asigna a recursos para organizar, filtrar, imputar costos o aplicar gobierno. Ejemplo: ambiente=prod.

Política (Azure Policy)

Mecanismo para imponer reglas de gobierno sobre recursos Azure. Puede auditar, bloquear o corregir despliegues que no cumplan con lineamientos.

Bloqueo de recursos (Resource Lock)

Mecanismo de protección para evitar eliminación o modificación accidental de recursos críticos. Puede ser de tipo Delete o ReadOnly.

RBAC

Modelo de control de acceso basado en roles. Define qué acciones puede hacer una identidad sobre un alcance como suscripción, grupo de recursos o recurso.

2. Identidad y acceso

Microsoft Entra ID

Servicio de identidad y acceso de Azure. Administra usuarios, grupos, aplicaciones, autenticación y tokens. Antes se conocía como Azure Active Directory.

Directorio activo

En contexto Azure suele referirse a Microsoft Entra ID como directorio de identidades en la nube, aunque Active Directory tradicional es el servicio on-premises de Windows Server.

Usuario

Identidad de persona o cuenta que puede autenticarse en el entorno y recibir permisos sobre recursos o aplicaciones.

Grupo

Conjunto de usuarios o identidades que facilita asignar permisos de forma colectiva en lugar de hacerlo uno por uno.

Rol

Conjunto predefinido o personalizado de permisos. Ejemplo: Reader, Contributor u Owner.

Permiso

Acción concreta autorizada, como leer, crear, actualizar o eliminar recursos, o bien leer secretos dentro de una bóveda.

Identidad administrada

Identidad de un recurso Azure en Entra ID para autenticarse contra otros servicios sin guardar credenciales en el código.

Managed Identity system-assigned

Identidad administrada ligada directamente a un recurso. Si el recurso se elimina, la identidad también desaparece.

Managed Identity user-assigned

Identidad administrada creada como recurso independiente y reutilizable por varios servicios.

RDP

Remote Desktop Protocol. Protocolo de acceso remoto gráfico usado normalmente para administrar servidores Windows.

SSH

Secure Shell. Protocolo de acceso remoto seguro por terminal, muy usado para servidores Linux.

3. Redes y conectividad

CIDR

Forma estándar de expresar bloques de direcciones IP, por ejemplo 10.0.0.0/16. Se usa para definir rangos de red y subredes.

Geografía

Agrupación amplia de regiones Azure dentro de un área geográfica, usada para residencia de datos, cumplimiento y planeación de continuidad.

Región

Ubicación específica de Azure donde se despliegan recursos, por ejemplo East US o West Europe.

Zona de disponibilidad

Ubicación física separada dentro de una región Azure, usada para mejorar resiliencia local frente a fallos de datacenter.

Par de regiones

Relación predefinida entre dos regiones Azure dentro de una misma geografía para apoyar continuidad operativa y orden de mantenimiento.

VNet (Virtual Network)

Red privada lógica dentro de Azure donde se conectan recursos y se define direccionamiento, segmentación y conectividad.

Subred

Segmento interno de una VNet. Permite separar cargas por función, seguridad o diseño de arquitectura.

NIC (Network Interface)

Interfaz de red de un recurso, como una VM. Es el componente que le da conectividad dentro de la red.

IP pública

Dirección accesible desde Internet. Se usa cuando un recurso debe ser alcanzable desde fuera de la red privada.

IP privada

Dirección interna dentro de una VNet o red privada. Sirve para comunicación interna entre servicios.

DNS

Sistema de resolución de nombres a direcciones IP. Permite que una aplicación use nombres legibles en lugar de memorizar IPs.

NAT

Network Address Translation. Permite que varios recursos privados salgan a Internet usando una o varias IPs públicas compartidas.

Application Gateway

Balanceador de aplicaciones y reverse proxy HTTP/HTTPS de capa 7 en Azure. Sirve para publicar aplicaciones web, terminar TLS, balancear carga y enrutar tráfico por dominio o ruta.

NSG (Network Security Group)

Conjunto de reglas de red para permitir o denegar tráfico de entrada y salida según origen, destino, puerto y protocolo.

UDR

User Defined Route. Ruta definida por el usuario para cambiar el siguiente salto del tráfico dentro de una red Azure.

NVA

Network Virtual Appliance. Appliance o dispositivo virtual de red, como firewall o equipo de inspección desplegado como máquina virtual.

Private Endpoint / Enlace privado

Mecanismo para acceder a un servicio PaaS mediante una IP privada dentro de tu VNet, evitando exponerlo a Internet.

Service Endpoint / Punto de conexión del servicio

Mecanismo que extiende la identidad de una subnet hacia un servicio Azure para que el tráfico permanezca en la red de Microsoft sin exponer el recurso a Internet pública.

Public Endpoint / Enlace público

Acceso a un servicio mediante su interfaz pública accesible desde Internet, normalmente protegido con firewall, autenticación u otras restricciones.

VPN Gateway

Servicio que conecta Azure con redes on-premises u otras redes Azure mediante túneles cifrados sobre Internet.

ExpressRoute

Conexión privada dedicada entre la red corporativa y Azure, con mayor previsibilidad y aislamiento que una VPN sobre Internet.

CORS

Cross-Origin Resource Sharing. Regla del navegador que controla desde qué orígenes web puede consumirse un recurso HTTP. No sustituye controles reales de red.

4. Datos y almacenamiento

Contenedor

Paquete ejecutable que incluye aplicación, dependencias y runtime. Permite que la app corra de forma consistente entre distintos entornos.

Pod

Unidad básica de ejecución en Kubernetes. Normalmente contiene uno o varios contenedores que comparten red y almacenamiento cercano.

Nodo

Máquina del clúster donde se ejecutan pods. En AKS normalmente corresponde a una máquina virtual de Azure.

Clúster

Conjunto coordinado de nodos y componentes de control que permite ejecutar aplicaciones distribuidas.

Kubernetes

Plataforma de orquestación de contenedores que automatiza despliegue, escalado, programación y recuperación de aplicaciones en contenedores.

Node pool

Grupo de nodos con configuraciones similares dentro de un clúster, útil para separar tipos de carga o sistemas operativos.

AKS

Azure Kubernetes Service. Servicio administrado de Azure para desplegar y operar clústeres de Kubernetes.

SMB / NFS

Protocolos de compartición de archivos en red. Azure Files puede exponer recursos compartidos usando estos protocolos.

SAS

Shared Access Signature. Token temporal que otorga acceso delimitado por tiempo, permisos y alcance a recursos de almacenamiento.

ADLS Gen2

Azure Data Lake Storage Gen2. Extensión de Storage Account orientada a analítica masiva, con jerarquía real de carpetas y capacidades de Data Lake.

Azure SQL Database

Servicio PaaS de base de datos relacional para cargas transaccionales y datos estructurados.

Azure SQL Managed Instance

Servicio PaaS con compatibilidad más cercana a una instancia tradicional de SQL Server, orientado a migraciones con menos cambios.

Storage Account

Cuenta de almacenamiento que puede ofrecer blobs, archivos, colas y tablas. Es la base de varios escenarios de datos no estructurados.

LRS / ZRS / GRS

Opciones de redundancia en Storage. LRS mantiene copias locales, ZRS replica entre zonas de disponibilidad y GRS agrega copia geográfica en otra región.

Data Lake

Repositorios de datos masivos y flexibles para analítica, históricos, ciencia de datos o integración a gran escala.

Azure Data Factory

Servicio de integración y orquestación de datos. Sirve para copiar, transformar y mover datos entre sistemas.

SQL Data Sync

Servicio legacy de sincronización de datos entre bases Azure SQL y SQL Server, con modelo de consistencia eventual y retiro anunciado.

T-SQL

Dialecto SQL de Microsoft SQL Server. Extiende SQL estándar con instrucciones, funciones y mecanismos propios del ecosistema Microsoft.

SQL Server Agent

Motor interno de jobs programados en SQL Server para ejecutar tareas automáticas como mantenimientos, procesos o cargas.

CLR

Capacidad de ejecutar código .NET dentro del motor SQL para extender funciones del servidor de base de datos.

OLTP

Online Transaction Processing. Patrón orientado a transacciones operativas del negocio, con muchas lecturas y escrituras rápidas y consistentes.

ETL / ELT

Patrones de integración de datos. ETL significa extraer, transformar y cargar; ELT significa extraer, cargar y transformar dentro del destino.

SSIS

SQL Server Integration Services. Tecnología de Microsoft para procesos de integración y movimiento de datos.

VMSS

Virtual Machine Scale Sets. Grupo administrado de múltiples máquinas virtuales que permite escalar horizontalmente.

5. Seguridad y cumplimiento

Key Vault

Servicio para almacenar secretos, certificados y claves criptográficas de forma segura y controlada.

Secreto (Secret)

Valor sensible como una contraseña, token, API key o connection string que no debe exponerse directamente en código o configuración.

Certificado

Objeto digital usado para autenticación, cifrado o HTTPS/TLS. Puede ser administrado en Key Vault.

WAF (Web Application Firewall)

Firewall de aplicaciones web. Inspecciona peticiones HTTP/HTTPS para detectar y bloquear ataques comunes como SQL injection o cross-site scripting. En Azure puede usarse con servicios como Application Gateway y Front Door.

TDE

Transparent Data Encryption. Mecanismo de cifrado en reposo para proteger archivos de base de datos, logs y respaldos.

Always Encrypted

Protección de columnas sensibles para que ciertos datos no sean visibles en claro incluso para administradores no autorizados.

Dynamic Data Masking

Ocultamiento parcial del valor mostrado a ciertos perfiles. Sirve para exposición controlada, no sustituye cifrado real.

WORM

Write Once, Read Many. Modo de inmutabilidad donde un dato puede escribirse una vez y luego solo leerse, sin modificarse ni borrarse durante el periodo definido.

PII

Personally Identifiable Information. Información personal identificable, como nombre, identificadores, correos o datos financieros asociados a una persona.

TLS

Transport Layer Security. Protocolo de cifrado para proteger comunicaciones en tránsito, por ejemplo entre una aplicación y una base de datos.

6. Monitoreo y operación

Azure Cost Management

Conjunto de herramientas para analizar, controlar y optimizar costos reales en Azure mediante vistas de consumo, presupuestos y alertas.

Billing

Ámbito de facturación donde se gestionan cargos, facturas, métodos de pago y estructuras como billing account, billing profile e invoice section.

Azure Monitor

Plataforma unificada de observabilidad en Azure para métricas, logs, alertas y diagnóstico de recursos.

Application Insights

Capacidad de monitoreo de aplicaciones dentro de Azure Monitor. Permite analizar requests, excepciones, dependencias y tiempos de respuesta.

APM

Application Performance Monitoring. Monitoreo de rendimiento de aplicaciones, orientado a tiempos de respuesta, errores, dependencias y experiencia real.

Log Analytics Workspace

Espacio de trabajo donde se almacenan y consultan logs con KQL en Azure Monitor.

KQL

Kusto Query Language. Lenguaje de consulta usado en Azure Monitor y Log Analytics para analizar logs y telemetría.

Azure Advisor

Servicio que entrega recomendaciones sobre costo, seguridad, confiabilidad, rendimiento y excelencia operativa.

Azure Service Health

Vista de incidentes, mantenimientos planeados y eventos de plataforma que pueden afectar tus suscripciones.

Azure Resource Health

Salud individual de un recurso específico. Ayuda a diagnosticar si un problema proviene del servicio Azure o de la configuración propia.

Alerta

Regla que se dispara cuando una métrica, log o evento de salud cumple una condición definida y activa una notificación o automatización.

Action Group

Conjunto reutilizable de acciones que Azure ejecuta cuando una alerta se activa. Un Action Group no detecta el problema; la alerta lo detecta y el Action Group define qué respuesta se dispara, por ejemplo enviar correo, SMS, webhook, notificaciones push o automatizaciones. Ejemplo: si una alerta detecta que la CPU de una VM supera 90%, el Action Group puede avisar al equipo y lanzar una acción correctiva.

DCR

Data Collection Rule. Regla de recolección de datos que define qué telemetría se captura, cómo se transforma y hacia dónde se envía.

Idea práctica: este glosario no sustituye las sesiones temáticas. Su función es dar una definición corta para ubicar cada concepto dentro de la arquitectura de Azure.
